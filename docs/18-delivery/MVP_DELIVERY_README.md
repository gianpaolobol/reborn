# MVP Delivery Pack

This folder explains how to use Step 3.

## What Step 3 adds

Step 3 converts the strategic documentation into an execution-ready product package.

It adds:

- MVP lock;
- epics;
- tickets;
- prioritised backlog;
- traceability matrix;
- user flows;
- textual wireframes;
- prototype script;
- implementation plan;
- route map;
- API endpoints;
- SQLite schema;
- seed data;
- sprint plan;
- release criteria;
- pilot plan;
- GitHub issue templates;
- issue creation script.

## How to use it

1. Push the repository to GitHub.
2. Review `docs/03-prd/MVP_LOCK_v0.1.md`.
3. Review `docs/03-prd/MVP_TICKETS.md`.
4. Create GitHub issues using `scripts/create-github-issues.ps1` or manually.
5. Build clickable prototype from `docs/06-wireframes/MVP_WIREFRAMES.md`.
6. Only after prototype review, start implementation from Sprint 1.

## Do not skip

Do not jump directly to backend before validating:

- repair journey;
- diagnosis summary;
- repair path comparison;
- provider quote flow;
- outcome feedback.

These screens are the core product.
