# Prototype Admin Operations UI

## Route

`#/ops`

## Purpose

The Admin Operations UI makes Re-born more usable and governable by exposing operational review queues, moderation actions and escalation status in the prototype.

## Main panels

- Ops control panel
- Review queue
- Escalation timeline
- Ops policy/SLA card

## Admin actions

From the prototype an admin can trigger:

- Create review item
- Assign to me
- Record moderation action
- Escalate
- Resolve

In mock mode, the same UI remains demonstrable without API access.

## UX principle

The UI must communicate that Re-born is governing real-world repair risk, not just displaying marketplace data.
