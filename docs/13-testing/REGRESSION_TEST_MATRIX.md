# Regression Test Matrix

## Purpose

Step 39 formalises the CI smoke suite as a regression test matrix.

The matrix is not only a list of scripts. It maps every smoke test to:

- the step it protects;
- the bounded context or platform area;
- the product capability;
- the strategic asset it defends;
- the release gate severity.

This turns the validated Step 38 smoke suite into release evidence for every future step.

## Canonical source

The canonical runtime matrix is generated by:

```text
scripts/ci-release-evidence.ps1
```

The CI output is written to:

```text
storage/logs/ci-regression-test-matrix.json
```

This file is uploaded by GitHub Actions as part of the `reborn-ci-release-evidence` artifact.

## Gate levels

### critical

A critical smoke script protects a core user, provider, maker, AI, fulfilment, revenue or reporting capability. Failure means the branch is not suitable for merge or demo without explicit review.

### release-blocking

A release-blocking smoke script protects platform readiness, operations, privacy, release management or partner onboarding. Failure blocks pilot/beta release unless the failure is documented as intentionally out of scope.

## Future step rule
