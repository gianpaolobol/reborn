# Prototype QA Checklist

## Manual checks

- [ ] `public/prototype/index.html` opens locally in a browser.
- [ ] Navigation works for all hash routes.
- [ ] Mobile menu opens and closes.
- [ ] Repair journey routes preserve a coherent user story.
- [ ] Provider selection changes selected card.
- [ ] Repair path selection changes selected card.
- [ ] Toast messages appear for prototype-only actions.
- [ ] Layout remains usable below 920 px viewport width.
- [ ] No external dependency is required.
- [ ] No Bootstrap/template classes are used.

## UX checks

- [ ] Landing page communicates Repair Intelligence, not STL marketplace.
- [ ] Intake starts from object/problem.
- [ ] Diagnosis explains confidence and limitations.
- [ ] Repair paths show cost, ETA, risk and impact.
- [ ] AI generation is framed as fallback with validation.
- [ ] Provider view includes constraints and quality checks.
- [ ] Maker view explains royalties from fulfilled repairs.
- [ ] Dashboard connects repairs, credits and sustainability.
- [ ] Ops view includes Graph, Trust and AI validation queues.

## Accessibility baseline

- [ ] Skip link exists.
- [ ] Buttons and links are keyboard reachable.
- [ ] Form fields have labels.
- [ ] Navigation has aria labels.
- [ ] Color is not the only indicator for selected state.
- [ ] Text contrast is acceptable for prototype review.
