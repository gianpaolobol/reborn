# Design System — Component Principles

## Button

Un bottone deve descrivere l'azione concreta, non un generico “continua”.

Esempi:

- “Analizza oggetto”
- “Conferma componente”
- “Richiedi preventivo locale”
- “Valida riparazione”

## Card

Le card principali sono:

- Repair Case Card
- Repair Path Card
- Part Card
- Model Card
- Provider Card
- Maker Card
- Repair DNA Card
- Repair Passport Card

## Upload Zone

Deve comunicare:

- cosa caricare;
- formati accettati;
- privacy;
- utilità dell'input;
- cosa succede dopo.

## Confidence UI

Ogni output AI deve mostrare:

- confidence score;
- spiegazione;
- possibilità di correggere;
- impatto sul percorso successivo.

## Trust Badge

I badge devono essere tecnici e verificabili:

- Official Part
- Community Validated
- Provider Verified
- High Repair Success
- AI Generated Draft
- Needs Validation
