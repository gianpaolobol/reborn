# Re-born Principles

## Product principles

1. Start from the broken object, not from the file.
2. Reduce uncertainty at every step.
3. Make repair feel possible even for non-technical users.
4. Use AI to guide, not to pretend certainty.
5. Turn every repair into reusable knowledge.
6. Reward useful contributions from makers and providers.
7. Prefer verified solutions over abundant but unreliable results.
8. Build trust before scale.

---

## UX principles

1. One screen, one job.
2. Every screen must answer: what happened, what is needed, what comes next.
3. Never expose marketplace complexity before the repair path is clear.
4. Use plain language first, technical language second.
5. Show confidence, uncertainty and missing data honestly.
6. Make the user feel accompanied, not evaluated.

---

## Technical principles

1. Keep domain logic outside controllers and views.
2. Use explicit application services.
3. Use repositories for persistence boundaries.
4. Use domain events for meaningful state changes.
5. Prepare for API-driven growth.
6. Avoid premature complexity, but do not compromise domain boundaries.
