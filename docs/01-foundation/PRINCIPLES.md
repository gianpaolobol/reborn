# Product Principles

1. L’utente vuole riparare, non imparare la stampa 3D.
2. La UX parte dall’evento reale: “si è rotto qualcosa”.
3. Ogni riparazione deve alimentare il Knowledge Graph.
4. Ogni modello deve avere versioni, compatibilità e affidabilità misurabile.
5. L’AI non sostituisce il maker: lo potenzia.
6. La produzione deve essere il più possibile locale.
7. Le aziende devono poter partecipare con cataloghi ufficiali, API e dashboard ESG.
8. Il valore non è solo la transazione, ma la conoscenza accumulata.
9. Non si scrive codice definitivo senza UX approvata.
10. Ogni funzione deve rafforzare almeno un asset strategico.
