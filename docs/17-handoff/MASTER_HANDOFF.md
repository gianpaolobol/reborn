# Master Handoff — Re-born

## Cosa è Re-born

Re-born è una Repair Intelligence Platform. Non è un repository STL, né un service di stampa 3D, né un marketplace generico.

## Vision

Costruire l’infrastruttura mondiale della riparazione distribuita.

## Principio UX

L’utente non vuole uno STL. Vuole che l’oggetto torni a funzionare.

## Stato completato

- Vision
- Business Model
- Personas
- Repair Journey Framework
- Repair Intelligence Engine
- Repair Knowledge Graph
- DDD
- Database schema
- API contract
- Repository bootstrap

## Prossimo lavoro

1. Completare UX Bible.
2. Completare PRD con user stories e criteri di accettazione.
3. Completare Design System.
4. Progettare UI ad alta fedeltà.

Non scrivere codice casuale. Ogni modifica deve essere collegata a PRD, UX o architettura.
