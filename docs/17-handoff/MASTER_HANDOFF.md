# Master Handoff — Re-born

## Obiettivo

Costruire **Re-born**, una **Repair Intelligence Platform** internazionale con missione:

> Allow anyone to repair anything.

La piattaforma deve permettere di identificare automaticamente prodotti/componenti tramite AI, trovare il percorso migliore di riparazione, recuperare o generare ricambi, produrli localmente tramite provider distribuiti e apprendere da ogni riparazione.

## Decisioni prese

1. Re-born non è marketplace STL: è Repair Intelligence Platform.
2. La UX parte da “Che cosa vuoi riparare?”.
3. Repair Journey, Repair DNA e Knowledge Graph confluiscono nel Repair Intelligence Engine™.
4. I motori sono Recognition, Knowledge, Decision, Learning e Trust.
5. Backend: PHP 8.3+, Clean Architecture, DDD, SOLID, Repository Pattern, Domain Events.
6. Frontend: HTML5, CSS3, Vanilla JavaScript.
7. Database: SQLite dev, MariaDB/MySQL prod.
8. No Bootstrap, no template acquistati, no gradient viola SaaS generici, no glassmorphism pesante.
9. Border radius massimo: 4px.
10. Ogni feature deve rafforzare almeno un asset strategico.

## Vincoli

- Repository GitHub come unica fonte di verità.
- Nessun codice prima di product/UX/architettura coerenti.
- Ogni schermata deve risolvere un problema reale.
- L'utente deve percepire una riparazione guidata, non un marketplace.
- Ogni riparazione deve alimentare Knowledge Graph e Learning Engine.
