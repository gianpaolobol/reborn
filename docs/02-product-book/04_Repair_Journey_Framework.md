# Product Book — Capitolo 4: Repair Journey Framework™

## Principio

Re-born non gestisce file: gestisce eventi di riparazione.

## Fasi

1. Break — l’oggetto si rompe.
2. Discover — l’utente cerca aiuto.
3. Identify — AI identifica prodotto e componente.
4. Knowledge — ricerca nel Knowledge Graph.
5. Decision — il sistema decide il percorso migliore.
6. Production — il ricambio viene prodotto.
7. Delivery — spedizione, ritiro, locker o Repair Café.
8. Install — tutorial, guida, istruzioni.
9. Validation — l’utente conferma se funziona.
10. Learning — feedback e dati aggiornano il Knowledge Graph.

## Differenza strategica

Per Hubs l’ordine finisce con la consegna. Per MakerWorld il ciclo finisce con il download. Per Re-born la riparazione è solo l’inizio dell’apprendimento.
