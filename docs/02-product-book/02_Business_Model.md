# Product Book — Capitolo 2: Business Model

## Tesi

Re-born non deve essere solo un marketplace. Deve essere un ecosistema economico in cui ogni passaggio del Repair Loop genera valore.

## Repair Loop™

Oggetto rotto → Foto/File → AI → Knowledge Graph → Maker → Provider → Pagamento → Spedizione → Feedback → Learning → Knowledge Graph migliorato.

## Revenue Streams

1. Commissione sulla stampa.
2. Royalty sul modello.
3. Marketplace CAD.
4. AI Premium.
5. Abbonamento Maker PRO.
6. Abbonamento Provider PRO.
7. Brand Portal Enterprise.
8. White Label.
9. API Economy.
10. Marketplace materiali.
11. Marketplace componenti.
12. Certificazioni.
13. Bounty.
14. Analytics e report ESG.
15. Donazioni ai maker.

## Commissioni indicative

- Privati: 5–8%
- Service professionali: 10–15%
- Enterprise/Brand Portal: SaaS mensile o annuale

## KPI primario

Objects Saved™: oggetti salvati dalla sostituzione.

## Flywheel

Più utenti → più richieste → più modelli → più provider → più riparazioni → più feedback → AI migliore → Knowledge Graph migliore → più aziende → più utenti.
