# Prototype Script

The clickable prototype should demonstrate one complete repair loop.

## Demo scenario

A user has a broken plastic knob from a small appliance.

The goal is not to show every possible screen. The goal is to make an investor, provider or maker understand the Re-born difference:

> the user does not search for an STL; the user starts a repair.

## Prototype path

1. Homepage
2. Start repair
3. Upload photos
4. Describe issue
5. Add details
6. Submit case
7. Case timeline
8. Diagnosis summary
9. Repair path comparison
10. Provider quote request
11. Provider quote response
12. Outcome confirmation
13. Impact/Knowledge Graph update

## Narrative beats

### Beat 1 — The problem is human

“This object is broken. I do not know the part name.”

### Beat 2 — Re-born structures the problem

Photos and plain language become Repair DNA.

### Beat 3 — Re-born proposes paths

The system does not dump search results. It explains what can happen next.

### Beat 4 — Ecosystem responds

Provider/maker/model paths appear as repair infrastructure, not a generic marketplace.

### Beat 5 — The system learns

Whether the repair succeeds or fails, the Knowledge Graph gets stronger.
