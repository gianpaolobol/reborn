# MVP Product Requirements Document

## Obiettivo MVP

Validare che utenti reali siano disposti a usare Re-born per riparare oggetti attraverso foto/file, ricambi community e provider locali.

## Scope MVP

### Incluso

- Registrazione/login.
- Creazione Repair Case.
- Upload immagini.
- AI job predisposto.
- Library ricambi.
- Upload modelli maker.
- Provider registration.
- Preventivi stampa.
- Ordini stampa.
- Wallet base.
- Royalty calcolate.
- Feedback riparazione.

### Escluso MVP

- White label completo.
- API pubblica monetizzata.
- AI proprietaria custom.
- App mobile nativa.
- AR installation.
- Enterprise analytics avanzato.

## Criteri di successo MVP

- Un utente crea un repair case in meno di 3 minuti.
- Un maker pubblica un modello in meno di 5 minuti.
- Un provider riceve e accetta un ordine.
- Un ordine genera split: provider + royalty + piattaforma.
- Un feedback aggiorna Repair Score o Objects Saved.
